Navigate to the Leaderboard section of your use case to compare submitted models, for example the [Breast Cancer Screening Use Case](https://ai.tracebloc.io/explore/ai-breast-cancer-screening-and-image-classification?tab=leaderboard).

## The Leaderboard

Vendors are ranked by the score of their best performing model. Scores are calculated using the evaluation metric you selected when defining the use case (e.g. Accuracy, F1, mAP). See the full list of [supported metrics per data type and task](/create-use-case/define#supported-metrics-per-data-type-and-task).

![Leaderboard](/images/Breast-cancer-screening-use-case-the-leaderboard.avif)

### Leaderboard Columns

| Column | Description |
|--------|-------------|
| Rank | Position based on the best model score. Lower rank = better performance. |
| Team / Vendor | The participant or team that submitted the model. |
| Score | The model's evaluation result on your test data, using the metric you defined. |
| Model Size | Size of the submitted model in parameters or MB. Useful for comparing efficiency. |
| Energy Consumption | Compute resources consumed during training. Helps assess cost-efficiency. |
| Submissions | Total number of models submitted by the team. |
| Remaining Budget | How much of the allocated compute budget (in PFLOPs) the team has left. |

### Actions You Can Take

From the use case view, you can:

- **Compare models** — Use the leaderboard to identify the best performing and most efficient models side by side.
- **Stop or adjust training** — Reduce or revoke a team's remaining compute budget to control costs or end a round of evaluation.
- **Contact participants** — Reach out to vendors or researchers directly to discuss results, request further iterations, or negotiate next steps.
- **Select a winner** — Once evaluation is complete, choose the best model based on score, model size, and resource efficiency.

## Supported Evaluation Metrics

The table below lists all evaluation metrics available. When defining a use case, you select one of these metrics to score and rank submitted models. Each metric uses either ascending (lower is better) or descending (higher is better) sort order.

| Metric | Description | Sort Order |
|--------|-------------|------------|
| F1 Score | Balances precision and recall. Useful for imbalanced datasets. | Descending |
| Loss | Quantifies prediction error. Lower is better. | Descending |
| Recall | Proportion of actual positives correctly identified. | Descending |
| Precision | Proportion of predicted positives that are actually positive. | Descending |
| Accuracy | Proportion of predictions that match ground truth. | Descending |
| Mean Average Precision (mAP) | Quality of ranked predictions. Common in object detection. | Descending |
| Mean Absolute Error (MAE) | Average error magnitude, in the same units as the target. | Descending |
| Intersection over Union (IoU) | Overlap between predicted and ground truth regions. | Descending |
| Dice Coefficient | Similarity between predicted and ground truth segmentation regions. | Descending |
| Matthews Correlation Coefficient (MCC) | Classification quality using all parts of the confusion matrix. Ranges from -1 to 1. | Descending |
| Cohen's Kappa | Agreement between predictions and ground truth, accounting for chance. | Descending |
| Specificity (True Negative Rate) | Proportion of actual negatives correctly identified. | Descending |
| AUC | Ability to distinguish between classes across all thresholds. | Descending |
| PCK (Percentage of Correct Keypoints) | Proportion of keypoints within a specified distance of ground truth. | Descending |
| R² (Coefficient of Determination) | Proportion of variance explained by the model. 1.0 = perfect fit. | Descending |
| Mean Squared Error (MSE) | Average squared difference between predicted and actual values. | Descending |
| Root Mean Squared Error (RMSE) | Square root of MSE. Same units as target variable. | Ascending |
| Mean Absolute Percentage Error (MAPE) | Average percentage difference between predicted and actual values. | Descending |
| Symmetric Mean Absolute Percentage Error (SMAPE) | Symmetric variant of MAPE. Handles values near zero better. | Descending |
| Median Absolute Percentage Error (MdAPE) | Median of absolute percentage errors. Robust to outliers. | Descending |
| Theil's U (U2 Statistic) | Forecast accuracy relative to a naive benchmark. Below 1.0 = better than baseline. | Descending |
| Max Error | Largest absolute difference between predicted and actual values. | Descending |
| Direction Accuracy | How often the model correctly predicts the direction of change. | Descending |
| Log Loss | Quality of probability estimates. Penalizes overconfident wrong predictions. | Descending |
| AUC-PR | Precision-recall balance across thresholds. Useful for imbalanced data. | Descending |
| AUC-ROC | Class separation across all thresholds. | Descending |
| Top-3 Accuracy | How often the true label is in the top 3 predictions. | Descending |
| Top-5 Accuracy | How often the true label is in the top 5 predictions. | Descending |
| Quadratic Weighted Kappa (QWK) | Agreement with quadratic penalty for larger disagreements. | Descending |
| Brier Score | Mean squared difference between predicted probabilities and outcomes. | Descending |
| Balanced Accuracy | Average recall across all classes. Each class weighted equally. | Descending |
| F-beta Score (beta = 0.5) | Precision-recall balance, emphasizing precision. | Descending |
| F-beta Score (beta = 2) | Precision-recall balance, emphasizing recall. | Descending |
| Hamming Loss | Fraction of labels incorrectly predicted. | Descending |
| Jaccard Score | Intersection over union of predicted and ground truth labels. | Descending |
| NPV (Negative Predictive Value) | Proportion of predicted negatives that are actually negative. | Descending |
| Micro F1 | F1 computed from globally aggregated TP, FP, and FN counts. | Descending |
| F1 Weighted | F1 per class, weighted by class support. | Descending |
| Gini Coefficient | Discriminatory power between classes. Related to AUC-ROC. | Descending |
| Normalized Gini | Gini scaled relative to a perfect model. Ranges from -1 to 1. | Descending |
| GIoU (Generalized IoU) | Extends IoU by penalizing non-overlapping predictions. Ranges from -1 to 1. | Descending |
| Mean Average Precision @ IoU 0.50 | mAP requiring at least 50% bounding box overlap. | Descending |
| Mean Average Precision @ IoU 0.75 | mAP requiring at least 75% bounding box overlap. | Descending |
| Mean Average Recall @ 1 Detection | Recall using only the top detection per image. | Descending |
| Mean Average Recall @ 10 Detections | Recall using up to 10 detections per image. | Descending |
| mAP per Class | Average Precision reported individually per class. | Descending |
| Mean Per Joint Position Error (MPJPE) | Average Euclidean distance between predicted and true joint positions. | Descending |
| Object Keypoint Similarity (OKS) | Keypoint similarity accounting for scale and localization uncertainty. | Descending |
| Visibility Accuracy | Correctness of predicted keypoint visibility status. | Descending |
| PCK@0.05 | Keypoints within 5% normalized distance from ground truth. Strictest variant. | Descending |
| PCK@0.10 | Keypoints within 10% normalized distance from ground truth. | Descending |
| PCK@0.20 | Keypoints within 20% normalized distance from ground truth. | Descending |
| PCK@0.30 | Keypoints within 30% normalized distance from ground truth. | Descending |
| PCK@0.50 | Keypoints within 50% normalized distance from ground truth. Most lenient variant. | Descending |
| Mean Intersection over Union (mIoU) | Average IoU across all segmentation classes. | Descending |
| Frequency Weighted IoU | IoU per class, weighted by class frequency. | Descending |
| Pixel Accuracy | Proportion of correctly classified pixels. | Descending |
| Mean Pixel Accuracy | Per-class pixel accuracy averaged equally. | Descending |
| Boundary IoU | IoU computed on boundary pixels only. | Descending |
| Boundary F1 Score | F1 computed on boundary pixels only. | Descending |
| Hausdorff Distance | Maximum distance between predicted and true boundaries. | Descending |
| Average Surface Distance (ASD) | Average distance between predicted and true boundary points. | Descending |
| Root Mean Squared Logarithmic Error (RMSLE) | Error on a log scale. Useful for targets spanning orders of magnitude. | Descending |
| Median Absolute Error (Median AE) | Median of absolute errors. Robust to outliers. | Descending |
| Explained Variance | Variance captured by the model, independent of bias. Up to 1.0. | Descending |
| Mean Bias Error (MBE) | Average prediction bias. Positive = overestimation, negative = underestimation. | Descending |
| Concordance Index (C-Index) | Agreement between predicted risk scores and observed event ordering. | Descending |
